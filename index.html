iOS Safari + VoiceOver: native date/time input issues — reproduction page

This page contains only standard HTML <input type="date"> and <input type="time"> elements, with no JavaScript frameworks. Use it to reproduce two long-standing VoiceOver issues on iOS Safari.

How to reproduce (VoiceOver on)

  1. Unwanted activation: Swipe right to move the VoiceOver cursor through the inputs below without double-tapping. Observe whether the native picker becomes active or is announced as expanded merely from cursor focus.
  2. Double-tap failure: Set the VoiceOver cursor on an input in Case 2 (wrapped in an element that has a click listener) and double-tap. Observe whether the native picker opens. Compare with Case 1.
  3. If a picker opens, also observe where the VoiceOver focus lands (it often lands on an unexpected control instead of the picker or staying on the input).
